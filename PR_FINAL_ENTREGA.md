Proyecto final: Instrucciones para la entrega final
1. Fechas importantes
Hasta 3 de mayo: Entrega final del proyecto (incluyendo memoria descriptiva).
2. Entrega del proyecto
La entrega del proyecto debe incluir:
 Una memoria del proyecto.
 El código fuente listo para desplegar.
 Un script SQL para crear las tablas de las bases de datos dejando la aplicación
lista para empezar a usar (sin datos).
 Un script SQL para poblar la base de datos con un conjunto de datos de prueba
detallados (varios usuarios, contenidos, etc.) pruebas que permitan ver cómo
sería el sistema cuando estuviese lleno de contenido.
El proyecto debe entregarse a través del campus virtual, como un único archivo
comprimido. Solamente una entrega por grupo. La entrega debe ser nombrada
siguiendo la siguiente notación: proyecto_g<numGrupo>.zip. Por ejemplo, el grupo 2
entregará el siguiente nombre de fichero: proyecto_g2.zip.
Asimismo, la aplicación deberá estar instalada y funcionando en el VPS que vuestro
grupo tiene asignado.
3. Memoria
El proyecto se debe acompañar de una memoria descriptiva, cuyo formato y extensión
también son libres. Debe cubrir los siguientes apartados:
 Descripción detallada de la aplicación: explicación detallada de lo que se
puede hacer con la aplicación, acompañado de capturas de pantalla. Debe
incluir las secuencias de tareas y explicar todo lo que se puede hacer. Se
recomienda aprovechar material de la práctica 1 actualizando los bocetos y
mejorando el texto.
 Arquitectura de la aplicación: diseño (vistas, scripts adicionales), diseño de la
base de datos y descripción de las principales decisiones de diseño. Se
recomienda aprovechar material de las prácticas 2 y 3. Incluir también las
novedades desde la práctica 3.

Como parte de la evaluación, todos los miembros del equipo deben rellenar la
autoevaluación en Simeg (http://radamanthis.fdi.ucm.es/simeg), valorando su trabajo

sobre las tecnologías empleadas. La autoevaluación se abrirá el 4 de mayo.
